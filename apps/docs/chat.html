AI Chat

Real-time streaming conversations with Claude 3.5 Sonnet. Chat remembers context across messages, supports markdown, and streams responses token by token via Server-Sent Events.

How Chat Works

When you send a message, BorjaxAI fetches the full conversation history from PostgreSQL, prepends it to your new message, and sends the combined context to Claude 3.5 Sonnet on AWS Bedrock. The response streams back through the FastAPI backend and the Hono gateway as SSE events, reaching your client in real time.

text
Client → POST /v1/chat/send
           ↓
       Hono Gateway  (auth, token check)
           ↓
       FastAPI AI Backend
           ↓  loads conversation history from PostgreSQL
       AWS Bedrock  (Claude 3.5 Sonnet)
           ↓  streams tokens
       SSE Response → Client

Sending Messages

POST to /v1/chat/send with your message and an optional conversation_id.

New conversation — pass "conversation_id": null:

javascript
const response = await fetch('https://api.borjaxai.com/v1/chat/send', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json',
    'Accept': 'text/event-stream',
  },
  body: JSON.stringify({
    message: 'Summarize the history of the internet.',
    conversation_id: null,
  }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();
let conversationId = null;

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const chunk = decoder.decode(value);
  for (const line of chunk.split('\n')) {
    if (!line.startsWith('data: ')) continue;
    const event = JSON.parse(line.slice(6));
    if (event.type === 'token') process.stdout.write(event.content);
    if (event.type === 'done') conversationId = event.conversation_id;
  }
}

Continue a conversation — pass the ID from the previous response:

javascript
body: JSON.stringify({
  message: 'Now give me 3 key takeaways from that summary.',
  conversation_id: conversationId,   // from the previous done event
})

Memory & Context

BorjaxAI automatically maintains full conversation context. Every message you send is saved to the conversation in PostgreSQL, and the entire history is sent to Claude on each request. This means Claude can reference anything said earlier in the thread — names, facts, preferences, code snippets.

⚠️
Context limits: Claude 3.5 Sonnet has a 200K token context window. For very long conversations, BorjaxAI may truncate the oldest messages to keep the context within limits. Recent messages are always preserved.

Available Tools

In standard chat mode, Claude can use these built-in capabilities:

CapabilityDescription
Web SearchClaude can search the web for up-to-date information when you ask about recent events or need current data.
CalculatorClaude performs mathematical calculations accurately without hallucinating results.
Code ExecutionClaude can write and reason about code in any popular language, with syntax highlighting in responses.
SummarizePaste long text and ask Claude to summarize, extract key points, or rewrite it.

Markdown in Responses

Claude's responses are rendered as full Markdown in the web app, including:

  • Headers (h1–h3) for structured answers
  • Code blocks with syntax highlighting for 40+ languages
  • Bullet and numbered lists for structured output
  • Bold/italic/inline code formatting
  • Tables for comparative information
  • Block quotes for citations

Conversation History

All conversations are saved automatically. Retrieve them via the API or browse them in the web app. Each conversation has an auto-generated title derived from the first message.

bash
# List your conversations
curl https://api.borjaxai.com/v1/chat/conversations?page=1&limit=20 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

# Get all messages in a conversation
curl https://api.borjaxai.com/v1/chat/conversations/CONVERSATION_ID \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
