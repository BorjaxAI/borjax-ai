Agent Builder

Design custom AI agents with specific roles, goals, backstories, and tool access. Agents run as background tasks via CrewAI and can be reused indefinitely.

What Is an Agent?

An agent is a configured AI persona with a specific role (who it is), goal (what it optimizes for), and backstory (the expertise it draws from). When you run an agent against a prompt, it uses those attributes to shape its reasoning, tone, and output quality.

Agents are powered by CrewAI under the hood. Each agent instance you create is saved to your account and can be run as many times as you like with different prompts.

Building Your First Agent

You can build an agent in the web app UI or via the API. Here's how to do it via the API:

bash
curl -X POST https://api.borjaxai.com/v1/agents \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "SEO Content Writer",
    "role": "Senior SEO Content Strategist",
    "goal": "Create highly engaging, SEO-optimized content that ranks well and converts readers.",
    "backstory": "You are an experienced content strategist with 8+ years writing for tech companies. You understand search intent, keyword strategy, and how to write content that both search engines and humans love. You always structure articles with clear headings, include data-driven insights, and end with a strong CTA.",
    "tools": ["web_search", "summarize"]
  }'

Response:

json
{
  "id": "agent_8a3f2d1e",
  "name": "SEO Content Writer",
  "role": "Senior SEO Content Strategist",
  "goal": "Create highly engaging, SEO-optimized content...",
  "tools": ["web_search", "summarize"],
  "created_at": "2026-05-03T12:00:00Z"
}

Field Reference

FieldRequiredDescription
nameYesA friendly name for your agent (e.g., "Market Researcher")
roleYesThe agent's professional title — shapes how it presents expertise
goalYesWhat the agent optimizes for — used by CrewAI to guide task execution
backstoryRecommendedRichly described background — dramatically improves output quality
toolsNoArray of tool names the agent can use (see Available Tools below)
verboseNoBoolean. If true, logs detailed reasoning steps (useful for debugging)

Running an Agent

Run any agent with any prompt. The agent uses its role, goal, and backstory to guide its output:

bash
curl -X POST https://api.borjaxai.com/v1/agents/agent_8a3f2d1e/run \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Write a 1000-word SEO-optimized blog post about the benefits of AI for small businesses. Target keyword: AI tools for small business."
  }'

The response is a task ID. Poll GET /v1/tasks/{task_id} for the result:

json
{ "task_id": "task_2b4f8e1a" }

Agent Best Practices

✅
Tips for great agent output:

Write a rich backstory. The more specific and detailed the backstory, the better Claude can embody the role. Include years of experience, specific industries, notable achievements, and work style.

Make the goal action-oriented. "Write compelling copy" is better than "be a copywriter." Goals should describe the outcome the agent optimizes for.

Match tools to the role. A researcher needs web_search. A code reviewer needs code. Don't add tools that aren't relevant — it can distract the agent.

Provide clear, specific prompts. Agents work best with detailed task descriptions. Include the desired format, length, tone, and audience in your run prompt.
