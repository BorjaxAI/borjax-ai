Background Tasks

Submit complex, long-running AI work to BorjaxAI's async Celery queue. Tasks run in the background — you don't wait. Poll for results when ready.

Tasks vs Chat

Chat is great for interactive, back-and-forth conversations. Background tasks are built for deep work — research that takes 2 minutes, a 3,000-word article, a full data analysis. Tasks run asynchronously so your application stays responsive.

ChatBackground Task
Interaction modelReal-time SSE streamSubmit → poll
Typical duration1–30 seconds15 seconds – 3 minutes
Best forQ&A, code help, conversationResearch, writing, analysis, scraping
ResultStreamed to clientStored, fetch when ready
Retry on failureManualAutomatic (Celery retry)

Task Types

TypeDescriptionExample Prompt
research Deep research using a Research + Analyst CrewAI crew. Searches web, synthesizes findings, writes a structured report. "Research the competitive landscape of AI coding assistants in 2026 and identify market gaps."
write Long-form content creation. Outlines, drafts, and polishes articles, blog posts, reports, or documentation. "Write a 1,500-word blog post about the benefits of serverless architecture for startups."
analyze Analyzes data, text, code, or documents. Provides structured insights, recommendations, and summaries. "Analyze the following financial data and identify trends, anomalies, and recommendations: [data]"
scrape Structured web data extraction. Browses a URL or set of URLs and extracts structured data to a specified format. "Scrape the top 10 results from a search for 'best project management tools 2026' and extract product names, pricing, and key features."
custom Run a custom agent you created via the Agent Builder. Pass your agent_id in the request. Any prompt — the agent's role, goal, and backstory define the behavior.

Submitting a Task

POST to /v1/tasks
Include the task type and your prompt. For custom tasks, also include your agent_id.
bash
curl -X POST https://api.borjaxai.com/v1/tasks \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "research",
    "prompt": "Research the top 5 AI trends shaping enterprise software in 2026.",
    "agent_id": null
  }'
Response (202 Accepted):
json
{
  "id": "task_7f3a1b2c",
  "type": "research",
  "status": "pending",
  "prompt": "Research the top 5 AI trends...",
  "created_at": "2026-05-03T12:00:00Z"
}
Poll for status
Check every 3–5 seconds. The task typically completes in 30–120 seconds.
bash
curl https://api.borjaxai.com/v1/tasks/task_7f3a1b2c \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
Read the result
When status === "completed", the result field contains the full output.
json
{
  "id": "task_7f3a1b2c",
  "type": "research",
  "status": "completed",
  "prompt": "Research the top 5 AI trends...",
  "result": "# Top 5 AI Trends in Enterprise Software (2026)\n\n## 1. Agentic Workflows...",
  "tokens_used": 4821,
  "created_at": "2026-05-03T12:00:00Z",
  "completed_at": "2026-05-03T12:01:47Z"
}

Task Status Lifecycle

text
  POST /tasks
      ↓
  ┌─────────┐
  │ pending │  ← In Celery queue, waiting for a worker
  └────┬────┘
       ↓ worker picks up
  ┌─────────┐
  │ running │  ← CrewAI crew is executing
  └────┬────┘
       ↓
  ┌─────────────────────────────────────┐
  │ completed  │  failed                │
  │ result set │  error message set     │
  └────────────────────────────────────-┘
StatusMeaning
pendingTask is queued and waiting for a Celery worker to pick it up.
runningA worker is actively executing the task. CrewAI agents are running.
completedTask finished successfully. Read the result field.
failedTask encountered an error. Read the error field for the reason.

Token Usage for Tasks

Background tasks consume more tokens than a simple chat message because they use multi-agent CrewAI crews. Each agent in the crew calls Claude independently.

Task TypeTypical Token Usage
research2,000 – 6,000 tokens
write1,500 – 4,000 tokens
analyze1,000 – 3,000 tokens
scrape1,500 – 5,000 tokens
customVaries by agent complexity
💡
Example prompts that work great as tasks:
• "Research and summarize the latest news about [topic] from the past 7 days."
• "Write a professional email to a prospective client introducing our product."
• "Analyze this dataset and identify the top 3 opportunities: [data]"
• "Create a 10-item FAQ for a SaaS product that does [description]."
