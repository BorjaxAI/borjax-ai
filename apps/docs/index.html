BorjaxAI Documentation

Welcome to the BorjaxAI platform documentation. BorjaxAI gives you a full AI team — chat, background tasks, and custom agents — all powered by Claude 3.5 Sonnet on AWS Bedrock.

Note: BorjaxAI is currently in public beta. APIs are stable but may evolve. Follow our changelog for updates.

What is BorjaxAI?

BorjaxAI is a SaaS platform that provides three core capabilities:

  • AI Chat — Real-time streaming chat with Claude 3.5 Sonnet with full conversation history
  • Background Tasks — Submit long-running AI tasks to a Celery queue and poll for results
  • Agent Builder — Design custom AI agents with roles and tools, combine into CrewAI crews

Architecture Overview

The platform consists of several components working together:

Quick Start

Get up and running with BorjaxAI in under 5 minutes.

1. Create an Account

Register via the API or the web dashboard:

curl -X POST https://api.borjax.ai/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "password": "your_password",
    "full_name": "Your Name"
  }'

Response:

{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "user": {
    "id": "uuid",
    "email": "you@example.com",
    "plan": "free",
    "tokens_used": 0,
    "tokens_limit": 10000
  }
}

2. Send Your First Chat Message

Chat with Claude via streaming SSE:

curl -X POST https://api.borjax.ai/v1/chat/send \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -H "Accept: text/event-stream" \
  -d '{
    "message": "Hello! What can you help me with?",
    "conversation_id": null
  }'

3. Submit a Background Task

curl -X POST https://api.borjax.ai/v1/tasks \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "research",
    "prompt": "Research the top AI trends for 2025",
    "agent_id": null
  }'

4. Poll for Task Results

curl https://api.borjax.ai/v1/tasks/{task_id} \
  -H "Authorization: Bearer YOUR_TOKEN"

Tip: Use the web dashboard at /app/ for a visual interface to all these features without writing any code.

Token Counting

Tokens are counted as follows:

  • Chat: Input tokens + output tokens per message
  • Tasks: All tokens consumed by the agent crew during execution
  • Agents: Tokens consumed during agent run

What happens when I hit my limit?

When you reach your monthly token limit, the API returns a 429 Too Many Requests error. Upgrade your plan via the billing dashboard to continue.

AI Chat

Stream real-time conversations with Claude 3.5 Sonnet. Conversations maintain context across messages.

How it Works

Chat uses Server-Sent Events (SSE) to stream responses token-by-token. Each message is saved to the conversation history, and the full history is sent to Claude on each request for full context.

Starting a Conversation

Send conversation_id: null to start a new conversation. The response includes a conversation_id to use in subsequent messages:

// SSE stream events:
data: {"type":"token","content":"Hello"}
data: {"type":"token","content":"! I"}
data: {"type":"token","content":" can"}
data: {"type":"done","conversation_id":"uuid","tokens_used":42}

Continuing a Conversation

Pass the conversation_id from a previous response to continue the thread:

POST /v1/chat/send
{
  "message": "What did you just say?",
  "conversation_id": "existing-conversation-uuid"
}

Listing Conversations

GET /v1/chat/conversations?page=1&limit=20

Background Tasks

Submit complex, long-running AI tasks to BorjaxAI's Celery-powered queue. Tasks run asynchronously so you don't have to wait.

Task Types

TypeDescriptionTypical Duration
researchResearch a topic using the Research CrewAI crew30–120s
writeWrite long-form content on a topic20–90s
analyzeAnalyze provided data or text15–60s
customRun a custom agent (requires agent_id)Varies

Task Lifecycle

Tasks move through these states:

  • pending — In queue, waiting to be picked up by a worker
  • running — Currently being executed by a Celery worker
  • completed — Finished successfully, result available
  • failed — Execution failed, error message available

Polling Pattern

// Submit task
const { id } = await fetch('/v1/tasks', { method:'POST', ... })

// Poll until done
const poll = async () => {
  const task = await fetch(`/v1/tasks/${id}`)
  if (task.status === 'completed') return task.result
  setTimeout(poll, 3000)
}

API Reference: Authentication

All API endpoints (except register and login) require a Bearer JWT token.

Register

POST /v1/auth/register

Create a new user account.

Request Body

{ "email": "string", "password": "string", "full_name": "string" }

Response 201

{ "access_token": "string", "token_type": "bearer", "user": { ... } }

Login

POST /v1/auth/login

Login with email and password.

{ "email": "string", "password": "string" }

Get Current User

GET /v1/auth/me Auth Required

Returns the current authenticated user with token usage.

{ "id": "uuid", "email": "string", "full_name": "string", "plan": "free|starter|pro|agency", "tokens_used": 0, "tokens_limit": 10000 }
